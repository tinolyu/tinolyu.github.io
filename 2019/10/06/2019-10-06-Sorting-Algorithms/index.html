Useful Sorting Functions

Less: Is item v less than w? 

private static boolean less(Comparable v, Comparable w)
{
	return v.compareTo(w) < 0;
}

Exchange: Swap item in array a[] at index i with the one at index j. 

private static void exch(Comparable[] a, int i, int j)  // NOTICE: the type of the array is Comparable.
{
	Comparable swap = a[i];
	a[i] = a[j];
	a[j] = a[i];
}

Insertion Sort

Well, that‚Äôs also easy‚Ä¶üòÄ

But you must know insertion sort performs well when the array is nearly sorted. In insertion sort, we just have to make sure that the part we have already scanned is sorted, the rest part we don‚Äôt care.
So in iteration i, as long as a[i] is smaller than its left, we swap it and its left item until it is no less then its left.

On average, insertion sort uses ~1/4N^2 compares and ~1/4N^2 exchanges. In the worst case, insertion sort uses ~1/2N^2 compares and ~1/2N^2 exchanges. But for partially sorted arrays, insertion sort runs in linear time.
üòÖ Sorry I always don‚Äôt know how to calculate the average situation, I prefer to use Big-O. And both selection sort and insertion sort are O(n^2). 

public static void sort(Comparable[] a)
{
	int N = a.length;
	for(int i = 0; i < N - 1; i++)
	{
		for(int j = i; j > 0; j--)
		{
			if(less(a[j], a[j -1])
				exch(a, j, j - 1);
			else break;
		}
	}
}

Improvement: Shell Sort

Why not make our array partially sorted and than use insertion sort?
ü§îShellsort: h-sort array for decreasing sequence of values of h.

Which increment sequence to use? 3x + 1 (1, 4, 13, 40, 121, 364, ‚Ä¶) 

public static void sort(Comparable[] a)
{
	int N = a.length;
	int h = 1;
	while(h < N / 3) h = 3 * h + 1;
	while(h >= 1)
	{
		for(int i = h; i < N; i++)
		{
			int j = i;
			while(j >= h && less(a[j], a[j - h]) 
				exch(a, j, j - h);
				j -= h;
			// for(int j = i; j >= h && less(a[j], a[j - h]) ; j-=h) exch(a, j, j - h);
		}
		h = h / 3;  // When h = 1, it's insertion sort.
	}
}

The worst-case number of compares used by shellsort with 3x+1 increment is O( N^(3/2) )

Merge Sort

As its name, there are two steps in the merge sort (omg, just TWO steps) - merge and sort.

STEP ONE: Merge

Consider this question, when there are two already sorted array, assigned a[i] and b[j], and we want to sort them into one big array. This is a simple question, you need two pointers and find which pointer is pointed to a smaller item, select this element and add this pointer. And also, we need extra space to store that element.
Now consider the same question in one single array.

We need to sort these two parts and get a full-sorted array. Given two sorted subarrays a[lo] to a[mid] and a[mid+1] to a[hi], replace with sorted subarray a[lo] to a[hi]. This is merge. 

private static void merge(Comparable[] a, Comparable[] aux, int lo, int mid, int hi)
{
	for(int k = lo; k <= hi; k++)
	{
		aux[k] = a[k];
	}
	int i = lo, j = mid + 1;
	for(int k = lo; k <= hi; k++)
	{
		if(i > mid) a[k] = a[j++];  // When all the items in the left part have been added to the final array.
		else if(j > hi) a[k] = a[i++]; // When all the items in the right part have been added to the final array.
		else if(less(aux[i], aux[j]) a[k] = a[i++];
		else a[k] = a[j++];
	}
}

STEP TWO: Sort


Merge sort uses recursion and I‚Äôm really not good at recursionüò£.

private static void sort(Comparable[] a, Comparable[] aux, int lo, int hi){
	if(lo == hi) return;
	int mid = lo + (hi - lo) / 2;
	sort(a, aux, lo, mid);
	sort(a, aux, mid + 1, hi);
	merge(a, aux, lo, mid, hi);
}

public static void sort(Comparable[] a)
{
	Comparable[] aux = new Comparable[a.length];  // You don't have to initialize aux each time.
	sort(a, aux, 0, a.length - 1);
}

Improvement

Merge sort uses at most NlgN compares and extra sapce proportional to N.

  1. Use insertion sort for small subarrays. Cutoff to insertion sort for ~7 items.
  2. Stop if already sorted. Is biggest item in first half <= smallest item in second half?
  3. Eliminate the copy to the auxiliary array. Save time, not space. Switching the role of the input and auxiliary array in each recursive call.

No recursive? Bottom-up mergesort

public static void sort(Comparable[] a)
{
	int N = a.length;
	Comparable[] a = new Comparable[a.length];
	for(int sz = 1; sz < N; sz = sz + sz)
	{
		for(int lo = 0; lo < N - sz + 1; lo += sz)
		{
			merge(a, aux, lo, lo + sz - 1, Math.min(lo + sz + sz - 1, N-1));
		}
	}
}

Quick Sort

STEP ONE: Shuffle the array


Make the array randomly sorted. For instance, if the array is increased, it would be inefficient, patitioning doesn‚Äôt do anything. Why not use insertion sort in this case?

*Knuth Shuffle: In iteration i, pick integer r between 0 and i uniformly at random.*

Public static void shuffle(Obeject[] a)
{
	int N = a.length;
	for(int i = 0; i < N; i++)
	{
		int r = StdRandom.uniform(i + 1);   
		// Generate random number rom the beginning to current location. NOTICE: not from 0 to N-1, but from i to N-1 is OK.
		exch(a, i , r);
	}
}

STEP TWO: Partition

private static int partition(Comparable[] a, int lo, int hi)
{
	int i = lo, j = hi + 1;
	while(true)
	{
		while(less(a[++i], a[lo]))
			if(i == hi) break;
		while(less(a[lo], a[--j]))
			if(j == lo) break;
			
		if(i >= j) break;
		exch(a, i , j);
	}
	exch(a, lo, j);
	return j;
}

STEP THREE: Sort

private static void sort(Comparable[] a, int lo, int hi)
{
	if(hi <= lo) return;  
	//So we can see if there is one element in this time of sorting, it will straightly return, this helps improve efficiency. 
	int j = partition(a, lo, hi);
	sort(a, lo, j - 1);  //Recursion
	sort(a, j + 1, hi);
}

Improvement : 3-way quicksort

When there are lots of duplicate elements in the array, using quick sort is not efficient.
üòà ~1/2N^2 compares when all keys are equal. Since each time, the partition doesn‚Äôt work and you have to iterate the other side. Using Dijkstra‚Äôs 3-way partitioning.

Dutch national flag problem: https://en.wikipedia.org/wiki/Dutch_national_flag_problem
üòÄ N*lgN when all distinct. 

private static void sort(Comparable[] a, int lo, int hi)
{
	if(hi <= lo) return;
	int lt = lo, gt = hi;
	Comparable v = a[lo];
	int i = lo;
	while(i <= gt)
	{
		int cmp = a[i].compareTo(v);
		if(cmp < 0) exch(a, lt++, i++);  // a[i] < v, exchange a[lt] with a[i], increment both lt and i
		if(cmp > 0) exch(a, i, gt--);  // a[i] > v, exchange a[gt] with a[i], decrement gt
		else i++;  // increment i
	}
	sort(a, lo, lt - 1);
	sort(a, gt + 1, hi);
}
